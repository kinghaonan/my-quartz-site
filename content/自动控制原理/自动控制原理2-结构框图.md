### 2.3.1 结构框图的基本概念

### 系统与子系统

**系统（System）**：按一定秩序或因果关系相互联系、相互作用和相互制约的一组事物（子系统）所构成的体系。

**子系统（Subsystem）**：构成系统的各个组成部分，一般采用具有输入、输出和传递函数的方框进行表示。
![[attachments/结构框图示例.png]]
### 结构框图定义

**结构框图/结构方框图（Block Diagram）**：描述控制系统各组成子系统间变量的传递和变换关系的图形表达式。

结构框图的特点：
1. **以传递函数概念为基础**：每个子系统用传递函数表示
2. **突出信息流而隐藏系统细节**：关注信号传递关系而非具体实现
3. **便于系统分析与设计**：直观展示系统结构，便于简化与分析

### 基本结构框图示例

典型的反馈控制系统结构框图：

$$
\begin{cases}
U(s) = R(s) \mp M(s) \\
Y(s) = G(s)U(s) \\
M(s) = H(s)Y(s)
\end{cases}
$$

图示表示：
```
R(s) → ⊕ → U(s) → [G(s)] → Y(s)
         ↑          ↓
         └──[H(s)]←─┘
```

其中：

- $R(s)$：参考输入/设定值
- $Y(s)$：系统输出
- $U(s)$：控制输入
- $M(s)$：反馈信号
- $G(s)$：前向通路传递函数
- $H(s)$：反馈通路传递函数

## 结构框图的四要素

结构框图由四个基本要素组成，每个要素都有特定的表示方法和含义。

### 方框（子系统/过程）

表示具有特定传递函数的子系统或过程：

```
R(s) → [G(s)] → C(s)
```

数学关系：
$$
C(s) = G(s)R(s)
$$

其中$G(s)$为子系统的传递函数。

### 有向线（信号）

表示信号传递的方向和路径：

```
r(t) → C(s)
```

有向线的特点：
- **方向性**：表示信号传递方向
- **连续性**：信号沿有向线连续传递
- **标注符号**：通常标注信号的名称或符号

### 求和点/比较点/综合点

对多个输入信号进行代数求和：

```
R₁(s) → ⊕ → C(s)
R₂(s) ↗
R₃(s) ↗
```

求和点规则：
1. **单输出**：求和点只有一个输出
2. **标极性**：每个输入信号需标注极性（+或-）
3. **代数求和**：输出等于各输入信号的代数和

数学表示：
$$
C(s) = R_1(s) + R_2(s) - R_3(s)
$$
![[attachments/求和点.png]]

### 引出点/传感点

从信号线中引出信号，用于多路传递：

```
R(s) → · → R(s)
        ↓
        R(s)
```

引出点规则：
1. **单输入**：引出点只有一个输入
2. **信号一致**：所有引出信号与原信号相同
3. **无能量损耗**：引出操作不改变信号大小

## 基本连接形式

### 串联形式/因果形式
![[attachments/串联.png]]

多个子系统按顺序连接，前一个子系统的输出作为后一个子系统的输入：

```
U(s) → [G₁(s)] → K(s) → [G₂(s)] → Y(s)
```

等效传递函数：
$$
G_{eq}(s) = G_1(s)G_2(s)
$$

数学推导：
$$
\begin{aligned}
K(s) &= G_1(s)U(s) \\
Y(s) &= G_2(s)K(s) = G_2(s)G_1(s)U(s) \\
\therefore G_{eq}(s) &= \frac{Y(s)}{U(s)} = G_1(s)G_2(s)
\end{aligned}
$$

### 并联形式

![[attachments/并联.png]]
多个子系统共享同一输入，输出求和：

```
          ↗ [G₁(s)] → Y₁(s)
U(s) → ⊕ → ↘ [G₂(s)] → Y₂(s)
         ↓
         Y(s) = Y₁(s) + Y₂(s)
```

等效传递函数：
$$
G_{eq}(s) = G_1(s) + G_2(s)
$$

数学推导：
$$
\begin{aligned}
Y_1(s) &= G_1(s)U(s) \\
Y_2(s) &= G_2(s)U(s) \\
Y(s) &= Y_1(s) + Y_2(s) = [G_1(s) + G_2(s)]U(s) \\
\therefore G_{eq}(s) &= \frac{Y(s)}{U(s)} = G_1(s) + G_2(s)
\end{aligned}
$$

### 反馈形式
![[attachments/反馈.png]]

输出信号通过反馈通路返回与输入信号比较：

```
R(s) → ⊕ → U(s) → [G(s)] → Y(s)
        ↑          ↓
        └──[H(s)]←─┘
```

反馈形式的数学关系：
1. **前向通路**：$G(s)$
2. **反馈通路**：$H(s)$
3. **求和点关系**：$U(s) = R(s) \mp M(s)$
4. **反馈信号**：$M(s) = H(s)Y(s)$
5. **输出关系**：$Y(s) = G(s)U(s)$

## 反馈类型与传递函数

### 正反馈与负反馈

**负反馈（Negative Feedback）**：
```
R(s) → ⊕ → U(s) → [G(s)] → Y(s)
        ↑ (-)      ↓
        └──[H(s)]←─┘
```
求和点：$U(s) = R(s) - M(s)$

**正反馈（Positive Feedback）**：
```
R(s) → ⊕ → U(s) → [G(s)] → Y(s)
        ↑ (+)      ↓
        └──[H(s)]←─┘
```
求和点：$U(s) = R(s) + M(s)$

### 闭环传递函数推导

#### 负反馈系统

由关系式：
$$
\begin{cases}
U(s) = R(s) - H(s)Y(s) \\
Y(s) = G(s)U(s)
\end{cases}
$$

消去$U(s)$：
$$
Y(s) = G(s)[R(s) - H(s)Y(s)]
$$
$$
Y(s) = G(s)R(s) - G(s)H(s)Y(s)
$$
$$
Y(s) + G(s)H(s)Y(s) = G(s)R(s)
$$
$$
Y(s)[1 + G(s)H(s)] = G(s)R(s)
$$

闭环传递函数：
$$
\Phi(s) = \frac{Y(s)}{R(s)} = \frac{G(s)}{1 + G(s)H(s)}
$$

#### 正反馈系统

同理可得：
$$
\Phi(s) = \frac{Y(s)}{R(s)} = \frac{G(s)}{1 - G(s)H(s)}
$$

### 通用反馈公式

对于一般反馈系统：
$$
\Phi(s) = \frac{\text{前向通路传递函数}}{1 \mp \text{回路传递函数}}
$$

其中：
- **负反馈**：分母取"$1 + G(s)H(s)$"
- **正反馈**：分母取"$1 - G(s)H(s)$"

## 子系统等价形式

### 串联系统的等价

两个串联子系统可等价为一个子系统：
```
U(s) → [G₁(s)] → [G₂(s)] → Y(s)
```
等价于：
```
U(s) → [G₁(s)G₂(s)] → Y(s)
```

### 并联系统的等价

两个并联子系统可等价为一个子系统：
```
          ↗ [G₁(s)] → 
U(s) → ⊕ → ↘ [G₂(s)] → Y(s)
```
等价于：
```
U(s) → [G₁(s)+G₂(s)] → Y(s)
```

### 反馈系统的等价

反馈系统可等价为单一方框：
```
R(s) → ⊕ → [G(s)] → Y(s)
        ↑          ↓
        └──[H(s)]←─┘
```
等价于：
```
R(s) → [G(s)/(1±G(s)H(s))] → Y(s)
```

## 结构框图的简化规则

### 基本简化原则

1. **串联简化**：串联子系统相乘
2. **并联简化**：并联子系统相加
3. **反馈简化**：应用反馈公式
4. **求和点移动**：相邻求和点可交换位置
5. **引出点移动**：相邻引出点可交换位置

### 求和点移动规则

#### 求和点前移（向输入端移动）

```
R(s) → ⊕ → [G(s)] → Y(s)
        ↑
     M(s)
```

等价于：
```
R(s) → [G(s)] → ⊕ → Y(s)
              ↑
           M(s)G(s)
```

数学关系：
$$
Y(s) = G(s)[R(s) + M(s)] = G(s)R(s) + G(s)M(s)
$$

#### 求和点后移（向输出端移动）

```
R(s) → [G(s)] → ⊕ → Y(s)
              ↑
           M(s)
```

等价于：
```
R(s) → ⊕ → [G(s)] → Y(s)
        ↑
     M(s)/G(s)
```

### 引出点移动规则

#### 引出点后移（向输入端移动）

```
R(s) → · → [G(s)] → Y(s)
        ↓
        R(s)
```

等价于：
```
R(s) → [G(s)] → · → Y(s)
                ↓
             R(s)/G(s)
```

#### 引出点前移（向输出端移动）

```
R(s) → [G(s)] → · → Y(s)
                ↓
                Y(s)
```

等价于：
```
R(s) → · → [G(s)] → Y(s)
        ↓
     R(s)G(s)
```

### 特殊规则

1. **不同类型点不可交换**：求和点与引出点不能直接交换位置
2. **移动需保持等价**：移动点后需调整相关传递函数
3. **避免交叉连接**：简化过程中尽量避免交叉连接

## 结构等价形式

### 基于叠加原理的等价转换

线性系统的叠加原理是结构等价转换的基础：

$$
\mathcal{L}\{a f_1(t) + b f_2(t)\} = a F_1(s) + b F_2(s)
$$

### 常见等价形式

##### 1. 求和点与方框的相对位置

```
U₁(s) → ⊕ → [G(s)] → Y(s)
U₂(s) ↗
```
等价于：
```
U₁(s) → [G(s)] → ⊕ → Y(s)
              ↑
           U₂(s)G(s)
```

##### 2. 引出点与方框的相对位置

```
R(s) → · → [G(s)] → Y(s)
        ↓
        R(s)
```
等价于：
```
R(s) → [G(s)] → · → Y(s)
                ↓
             Y(s)/G(s)
```

### 2.3.8 示例与解题方法

### 示例 2.3.1：多输入系统

考虑具有参考输入$R(s)$和扰动输入$N(s)$的系统：

```
R(s) → ⊕ → [G₁(s)] → ⊕ → [G₂(s)] → Y(s)
        ↑ (-)        ↑ (+)
        └──[H₁(s)]←─┘    N(s)
```

**求解步骤**：
1. 分别考虑$R(s)$和$N(s)$的作用（叠加原理）
2. 应用反馈公式简化内环
3. 处理外环反馈
4. 综合得到总传递函数

### 示例 2.3.2：复杂结构框图简化

```
R(s) → ⊕ → [G₁(s)] → ⊕ → [G₂(s)] → ⊕ → [G₃(s)] → C(s)
        ↑ (-)        ↑ (-)        ↑ (+)
        │            │            │
        ├─[H₁(s)]←───┤            │
        │            └─[H₂(s)]←───┤
        └────────────[H₃(s)]←─────┘
```

**简化策略**：
1. 从内向外逐步简化
2. 先处理局部反馈回路
3. 移动求和点和引出点以消除交叉
4. 应用串联、并联、反馈简化规则

### 解题技巧总结

1. **识别基本结构**：寻找串联、并联、反馈基本单元
2. **从内向外简化**：先简化内层结构，再处理外层
3. **移动点消除交叉**：通过移动求和点/引出点简化结构
4. **利用叠加原理**：多输入系统分别处理再叠加
5. **验证结果**：检查简化后的传递函数是否合理

        ↑ (-)        ↑ (-)
        │            │

### 2.3.10 图示法的意义与应用

### 三种系统表示方法的关系

控制系统有三种等价的数学表示方法：

1. **微分方程**：时域描述，物理意义明确
2. **传递函数**：频域描述，便于分析稳定性
3. **结构框图**：图示描述，直观展示系统结构

三者关系：
```
微分方程 ⇌ 传递函数 ⇌ 结构框图
```

### 结构框图的优势

1. **直观性**：清晰展示系统组成和信号流向
2. **模块化**：便于分系统设计和分析
3. **简化性**：通过规则简化复杂系统
4. **设计导向**：便于控制器设计和系统改进

### 工程应用

1. **自下向上建模**：从子系统开始，逐步构建完整系统模型
2. **控制系统设计**：明确受控对象、控制器、反馈通路
3. **系统性能分析**：通过简化分析系统稳定性、动态特性
4. **故障诊断**：通过结构分析定位问题环节

## 总结

### 核心要点

1. **四要素掌握**：方框、有向线、求和点、引出点的表示与含义
2. **三种基本形式**：串联、并联、反馈的连接与简化
3. **简化规则**：串联乘、并联加、反馈公式的应用
4. **等价转换**：基于叠加原理的点与框位置调整
5. **解题方法**：从内向外、移动点消除交叉的系统简化策略

### 重要公式

1. 串联等效：$G_{eq}(s) = G_1(s)G_2(s)$
2. 并联等效：$G_{eq}(s) = G_1(s) + G_2(s)$
3. 负反馈：$\Phi(s) = \frac{G(s)}{1 + G(s)H(s)}$
4. 正反馈：$\Phi(s) = \frac{G(s)}{1 - G(s)H(s)}$

### 学习建议

1. **多做练习**：通过大量练习掌握简化技巧
2. **理解原理**：深入理解叠加原理和代数关系
3. **直观思维**：培养从框图直观理解系统能力
4. **系统思维**：建立从局部到整体的系统分析思路

> 结构框图是控制系统分析与设计的重要工具，掌握其表示方法和简化规则对于后续的频率响应分析、根轨迹分析和控制系统设计具有重要意义。
